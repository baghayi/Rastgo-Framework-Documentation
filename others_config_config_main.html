بخش های دیگر

این فایل هم فایلیه که تنظیماتش برای هر پروژه ای ممکنه تعییر بکنه و البته یکی از ثابت هاش باید تغییر بکنه

ثابت ها

ثابت اولی SCRIPT_ROOT_FOLDER_NAME هستش که اگه پروژه داخل یه پوشه ای اجرا می شه باید اسم اون پوشه برای این ثابت وارد بشه

توجه کنید که اگه از دامین اصلی و یا سابدامین استفاده می کنید این تابت رو باید خالی بزارید و نیازی نیست پرش کنید (چیزی واردش کنید)

منظور از اینکه پوشه باشه اسم پوشه رو وارد کنید یعنی اینطوری:

http://test.com/project/
http://subdomain.test.com/project/

که برای هر دوی این موارد باید project رو در ثابت وارد کنید ولی اگر اینطوری باشه: (یعنی توپوشه نباشه)

http://test.com/
http://subdomain.test.com/

نیازی نیست تو این ثابت جیزی بنویسید و اگه چیزی هم باشه مقدارش رو خالی کنید: اینطوری ''

که البته بطور پیش فرض نام فریمورک برای این ثابت تعریف شده: RastgoFramework

ثابت بعدی هم WEB_PROTOCOL هستش و پورتکل پیشفرض رو تعریف میکنید

مثل اگه از http استفاده می کنید می نویسید http و برای https و ... هم همون رو می نویسید که بوطر پیش فرض هم http هست

ثابت بعدی هم TEMPLATE_FOLDER_NAME هستش که نام پوشه قالب تون رو می نویسید که قالب پیش فرض اسمش defaultTemplate هست و همین مقدار بطور پیش فرض به این ثابت داده شده

که از این ثابت برای تغییر قالب سایت می شه استفاده کرد

ثابت Port_Number_In_URL برای این هستش که شماره پورت رو به آخر آدرس مون اضافه کنه . مثلا وقتی از IP بجای آدرس دامین استفاده می کنید و می خوایید شماره پورت هم تو آدرس قراره بگیره می تونید این ثابت رو True تعیین کنید و در غیر اینصورت False.

ثابت بعدی Time_Zone و همونطور که از اسمش مشخصه برای تعیین time zone استفاده می شه و بطور پیش فرض مقدار Asia/Tehran رو داره.

ثابت Default_Controller هم برای تعیین کنترلر پیش فرض هستش که مقدار پیش فرض index هست و در صورتی که برنامه اجرا بشه سیستم دنبال کنترلر index خواهد گشت و اجرا می کنه و در صورتی که بخواهید کنترلر پیش فرض به اسم دیگه ای باشه می تونید توسط این ثابت این مورد رو تغییر بدید.

ثابت Default_Method هم مثل تقریبا مثل ثابت قبلی بوده، با این تفاوت که متد پیش فرض رو تغییر می ده که متد پیش فرض هم بازم index هست که با استفاده از این ثابت می تونید متد پیش فرض رو هم به اسم دلخواهتون تغییر بدید.

ثابت Enable_Rewrite که بصورت پیشفرض False‌ هست و باعث می شه لینک ها بصورت زیر باشن:

http://URL/?q=controller/method/Args/

و در اینصورت فریمورک می شه گفت وابسته به وب سرور خاصلی نیست و در صورتی که مقدار این ثابت رو True‌قرار بدیم لینک ها بصورتRewrite‌شده و مثل شکل زیر خواهند بود:

http://URL/controller/method/Args/

و در اینصورت وابسته به وب سرور (آپاچی) می شه. و در عوض لینک های User Friendly ای خواهیم داشت.

ناگفته نمونه که برای اینکه کل لینک ها (لینک هایی که توسعه دهنده بعدا به سیستم اضافه می کنه) هم تغییر کنن (از حالت اولی به دومی یا برعکس) نیازه که برای تعیین لینک ها از متد go کلاس Request استفاده بشه (چه برای redirect‌ کاربر و چه برای دریافت لینک بصورت String که در جاهای دیگه ای استفاده بشه) در غیر اینصورت چون لینک ها دستی تعیین می شه در نتیجه با تغییر مقدار این ثابت تغییری هم در لینک های ایجاده شده نخواهد بود!
