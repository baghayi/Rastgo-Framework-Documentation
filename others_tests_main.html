بخش های دیگر

این پوشه شامل فایل های تستی که برای بخش های مختلف فریمورک نوشته شدن هست و البته ممکنه فایل های تست برای همه بخش ها نباشه که به مرور زمان تا جایی که امکانش باشه آپدیت خواهند شد.

این تست ها توسط phpunit نوشته شدن و نیازه که برای اجراشون phpunit در سیستم نصب باشه.

همچنین این تست ها از autoloader مربوط به composer استفاده می کنه تا کلاس ها رو بتونه لود کنه و فایل composer.json هم به همین خاطر داخل پوشه هست که واسه همین خاطر نیازه که composer رو هم نصب داشته باشید که واسه اطلاعات بیشتر به سایت خود composer مراجعه کنید.

که با اجرای دستور composer install داخل پوشه Tests می تونید autoloader رو توسط composer درست کرده و در نتیجه فایل های تست برای اجرا شدن آماده هستن.

فایل phpunit.xml.dist هم شامل تنظیمات ای واسه phpunit هست که بتونه autoloader رو که در فایل bootStrap.php لود شده رو لود کنه و همینطور به فایل های تست هم دسترسی داشت باشه که با جرای دستور phpunit داخل پوشه تست، فایل phpunit.xml.dist توسط phpunit خواهند می شه و فایل های تست هم اجرا می شن.
