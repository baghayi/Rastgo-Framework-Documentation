هسته فریمورک

بخش های دیگر

این فایل وظیفه نگهداری تنظیماتی که توسعه دهنده بهش می ده رو داره و کلاس های مربوط بهش (LibraryController.php) از این فایل تنظیمات استفاده خواهد کرد تا طبق موارد تعیین شده درش عمل بشه و در حقیقت کلاس های خواسته شده لود بشه و شئ کلاس مورد نظر در اختیار توسعه دهنده در سراسر فریمورک قرار بگیره.

این فایل تنظیمات، شامل حداقل یک بخش و بیشتر، بسته به نیاز توسعه دهند و همینطور نیاز کلاس ای که قراره لود بشه داره.

در بخش اول نام کلاس/کلاس ها و همینطور نام/نام هایی که به عنوان property باید تعیین بشه و در حقیقت این property از طریق کلاس Registry قابل دسترسی خواهد بود.

یعنی شی کلاس مورد نظر توسط property ای که تعیین می شه قابل دسترسی خواهد بود که این proprty هم از طریق کلاس Registry قابل دسترسی خواهد بود.

اسم بخش اول به شرح زیر هست:

[Properties]

و مقادیر بعد از این تکه خط باید در فایل قرار بگیرن.

که بطور پیش فرض ۲ کلاس صدا زده شدن که به شرح زیر هست:

error = "ErrorReporting"
db = "DatabaseConfig"

که ErrorReporting و DatabaseConfig نام کلاسی هست که باید لود بشه هستن همچنین error و db هم اسمی که به عنوان property تعیین شده که شئ کلاس های مربوطه از این property ها قابل دسترسی باشن.

به عنوان مثال اگه بخواهیم در طول برنامه به شي کلاس های تعیین شده دسترسی داشته باشیم به شیوه زیر می شه عمل کرد:

global $registry;
$registry->error->...;
$registry->db->...;

بخش های بعدی که مورد نیاز خواهد بود و محدودیتی هم درش نیست که چند تا باشن برای ارسال مقادیر به متد سازنده __construct کلاس های فوق که تعیین شده اند استفاده می شن.

در حقیقت کلاس هایی که تعیین می شن تا شئ شون قابل دسترسی باشه، ممکنه پارامتر هایی در متد سازنده داشته باشن که نیاز باشه مقادیری بهشون ارسال بشه که از این بخش ها می شه برای این امر استفاده کرد.

که اسم بخش ها به شرح زیر خواهد بود:

[Argument:N]

که بجای N باید شماره قرار بگیره و به ترتیب از 1 تا ... خواهد بود و نشانگر پارامتر ای که باید مقدار/مقادیر بهش ارسال بشن هست.

که به عنوان مثال، ممکنه متد سازنده کلاسی ۳ پارامتر داشته باشه که در این صورت بخش هایی که باید ایچاد بشن به شرح زیر خواهد بود:

				[Argument:1]
...

[Argument:2]
...

[Argument:3]
...

که بجای ... (سه نقطه) هم باید مقدار/مقادیری که به پارامتر ارسال بشن قرار بگیره که نحوه تعیین مقدار/مقادیر جلوتر توضیح داده شده است.

قبل از اینکه به توضیح ... (سه نقطه) بپردازیم، یه تعریفی برای مقادیر تعیین می کنیم که از روی اون توضیح بدیم / به عنوان مثال به کد زیر نگاه کنید:

				[Argument:N]
x = y

که از این برای جایگاه x بهش خواهیم گفت کلید (KEY) و بجای جایگاه Y هم بهش می گیم مقدار (VALUE) که در طی توضیحات به این کلمات مواجه شدید، متوجه بشید منظور کدوم بخش هست.

که تا به اینجا متوجه شدیم که برای هر پارامتر یک بخش باید ایجاد بشه.

و برای ارسال مقادیر به کلاس مربوطه، باید در هر بخش (که مشخش کننده اینه که مقادیر به کدوم پارامتر ارسال بشه) بجای KEY نام property ای که در بخش ابتدایی به کلاس تعیین شده رو وارد کنید و برای value هم، مقداری که می خواهید ارسال بشه رو باید وارد کنید.

که در صورتی که می خواهید مقدار ارسال شده بصورت string باشه، باید به این طریق عمل کنید:

[Properties]
test = "TestClass"

[Argument:1]
test = 'The value that you want to be sent to the first argument'

که در این مثال، کلاس ای به اسم TestClass داریم که شئ کلاس در property ای بنام test قرار میگیره و برای اینکه string ای رو به پارامتر اولش ارسال کنیم در قسمت [Argument:1] کلید رو برابر test تعیین کردیم (که باید برابر نام ای property باشه) و مقدار هم یک متنی هست که باید بهش ارسال بشه.

که برای ارسال integer یا همان عدد هم به همین شیوه فوق عمل می کنید به اینصورت که نیازی نیست حتما کوتیشن ها رو ( ' یا " ) رو قرار بدید.

و در صورتی که بخواهید آرایه ای به پارامتر مربوطه ارسال کنید، به این شیوه عمل خواهید کرد:

[Properties]
test = "TestClass"

[Argument:1]
test[] = 'My name is Hossein'
test[] = 'My last name is Baghayi'
test[] = 21

که همونطوری که می بینید مثل php که آرایه رو به این شکل هم می شه تعریف کرد و مقدارش رو هم بهش می دیم که البته اینکه key مون چند بار تکرار بشه (یا به عبارتی، آرایه مون چند تا element داشته باشه) هم محدودیتی وجود نداره.

که یه مثال زیر رو هم ببینید:

[Properties]
test = "TestClass"
page = "Pagination"

[Argument:1]
test[] = 'My name is Hossein'
test[] = 'My last name is Baghayi'
test[] = 21
page = 50

[Argument:2]
page = 'Hello World'

که در این مثال هم دو کلاس برای property هاای test و page تعیین شده است و به کلاس testClass فقط به پارانتر اولش مقدار ارسال می شه که آرایه هست ولی به کلاس Pagination هم به پارامتر اولش مقدار ارسال می شه که عدد هست و هم به پارامتر دومش مقدار ارسال می شه که یک متن (string) هست.

و اینکه چند بخش برای پارامتر ایجاد کنیم هم محدودیتی درشت نیست و هم ارایه می شه تعیین کرد و هم عدد و هم رشته.

و مورد دیگه ای که هست، اینه که بجای value می تونید حتی ثابت هایی که خودتون تعریف کردید و یا از ثابت های خود PHP هم استفاده کنید! و مشکلی از این لحاظ نخواهد بود.

در کل، به این فایل، مثل باقی فایل های ini رفتار می کنید و فقط این بخش ها، که اسم هاشون چی باشن، مهم هست که باید بهشون توجه کرد.

توجه کنید که اگه کلاسی، پارامتری داشته باشه که اختیاری باشه، یعنی ضرورتا نیاز نیست بهش مقداری ارسال کنید که در اینصورت اگه مقداری هم بهش وارد نکنید، مشکلی نخواهد بود.

ولی اگه پارامتر ای باشه که نیاز داشته باشه بهش مقداری ارسال بشه (یعنی مقدار defalut/پیشفرض) نداشته باشه، در اینصورت باید حتما مقداری رو براش از این روش فوق ارسال کنید در غیر اینصورت خطا دریافت خواهید کرد.

و به این نکنه هم توجه کنید که اگه متد سازنده کلاسی چند پارامتر داشته باشه و نخواهید به پارامتر اول مقدار بدید و مستقیم برید به پارامترهای بعدی مقدار بدید، در اینصورت هم باز خطا خواهید گرفت که این مورد هم در خود PHP هم صادقه و فرقی نمی کنه.
